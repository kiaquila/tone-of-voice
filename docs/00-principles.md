# Principles

## Purpose

This repository exists to make author-voice support durable, inspectable, and updateable.

## Core Rules

- Treat voice memory as a living system, not a single prompt.
- Separate stable invariants from short-term observations.
- Prefer curated examples over large unstructured dumps.
- Keep platform adaptation explicit.
- Update summaries with evidence from published posts, not vibes.
- Preserve the author's sharpness, warmth, and point of view.

## Practical Standard

Every useful output from this repo should make it easier to answer:

- How does the author usually sound?
- What should stay constant across platforms?
- What should change for Telegram, Threads, and LinkedIn?
- Which examples best represent the current voice?
- What new material changed the profile recently?
