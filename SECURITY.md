# Security Policy

## Reporting a vulnerability

Please report security issues privately, never via a public issue or pull request:

1. Open the **Security** tab of this repository.
2. Click **Report a vulnerability** to file a private advisory.

If GitHub private vulnerability reporting is not available for you, contact the maintainer through the email address visible in commit metadata.

When reporting, include:

- a clear description of the issue and its impact
- a minimal reproducer or proof of concept
- affected commit SHA or branch
- any suggested mitigation

## Response expectations

This is a personal project. Responses are best-effort and there are no formal response or disclosure timelines. The maintainer will acknowledge receipt as soon as practical and coordinate a fix or mitigation before any public disclosure.

## Supported versions

Only the `main` branch is supported. There are no tagged releases.

## Out of scope

- Findings that require physical access to the maintainer's host or local credentials are out of scope.
- Theoretical issues without a working reproducer may be closed without action.
